# 执行流验证门 — 变异分类学（honest mutation taxonomy）

本门是**必要条件、非充分条件的宏观局部 stub 拦截器**。它的契约是：**零误报（绝不误挡忠实代码）优先；非宏观结构性的漏报是设计内允许的**（下游对拍兜底）。本文档诚实列出它能抓和故意不抓的变异类。

## 必抓（must-FAIL）——宏观结构/对外操作缺失

这些是门控的核心命中域。以下变异**必须 FAIL**：

| 变异类 | 示例 | 检测机制 |
|--------|------|----------|
| 删 CALL（丢一个可解析的调用目标） | 注释掉 `CALL(0x5F5C60)` | STEP3(0) 函数级 presence 必要条件 + 双射块内 CALL 集合差异 |
| 删 WRITE（丢一个可解析的成员/全局写） | 注释掉 `WRITE(member,1794)` | 同上 |
| 截断函数 / 删一条分支 | 整个 else 分支被注释掉 | 双射后继集合对不上 → structural MISMATCH |
| 结构性缺块 | C++ 侧少一个基本块 | block count 不匹配 or block_match 无配对 |

## 设计内抓不到（designed-FN, intentionally not caught）

这些是**刻意放弃的细粒度变异**。**不在本门范围内**，由下游对拍管线兜底：

| 变异类 | 示例 | 为什么故意不抓 | 兜底机制 |
|--------|------|---------------|----------|
| 改变分支比较运算符 | `if (x < 0)` → `if (x <= 0)` | 运算符与常量不编码为事件；条件不为本门建模 | 下游对拍（输出层 comparisonResult.log / 忒修斯之船） |
| 改变分支常量 | `if (x < 0)` → `if (x < 100)` | 同上 | 同上 |
| 交换操作数 | `this->A > this->B` → `this->B > this->A` | 同上 | 同上 |
| 改对象身份（同偏移不同对象） | `this->X = 1` → `target->X = 1`（偏移相同） | 当前大量 `void*`/`uint32_t*` 真实类型未确认，逐类深挖成本极高 | 同上 |
| 块内 CALL/WRITE 重排 | 块内两个调用的先后次序交换 | v2 块内集合比较（SET, not ordered）；块内定序是设计内放弃的 | 同上 |
| 变量下标成员访问 | `this->Array[i]`（无静态偏移） | 两边都不编码为固定偏移事件 | 同上 |
| 块内 CALL/WRITE 次数差异 | IDA 调某函数 2 次，C++ 只调 1 次 | v2 SET 比较（presence-only, not multiset）；次数级差异故意不抓 | 同上 |
| 大函数超时下的次数级差异 | 与上同 + 双射超时 | STEP3(0) 只检查 presence（≥1×）；次数级差异在超时路径下无法由双射抓（双射被跳过） | 同上 |

## 过渡期已知窗（transitional windows, closed by P2）

| 窗 | 何时闭合 | 当前处置 |
|----|---------|---------|
| 未解析成员（无 signals.json 偏移）→ 误报风险 | P2 地址映射正确后不存在未解析成员 | 元素级豁免 + WARN + offset-TODO；零误报（不 FAIL） |
| 偏移错位（`class_layouts.json` 偏移与 IDA 二进制不符，如 ObjectTypeClass=596 vs 真值 660）→ 该类的函数集体误报 | P2 对照 IDA 二进制核准 | P2 是硬前提——正确后不存在 |
| 超时/过大的函数双射放弃后次数级漏报 | 双射不超时时次数仍被 block_match 的 SET 比较抓 | 超时下 STEP3(0) 兜底 presence（不兜次数） |

## 下游兜底说明

本门的漏报**不是无人管的洞**。项目有**两层下游行为网**：

- **输出层面对拍**：Syringe 注入的 hook_dll 在同一状态上跑 RE 函数与原版函数，比对返回值与对外效果，差异记入 `comparisonResult.log`。专抓本门故意不管的细粒度错误。
- **实际效果层面（忒修斯之船）**：逐步用 RE 函数彻底取代原版函数，运行中的游戏本身成为测试。

> **诚实声明**：下游输出层面对拍**本身也受输入覆盖限制**（只有当玩法触及某边界，才会暴露该边界的运算符/常量错），**不是绝对保证**——它是概率性兜底。本门 + 对拍 + 游戏级替换 = 分层防御，各有分工。

## 误报测试（should-PASS）

以下变异**必须 PASS**（是忠实等价形式，挡它是误报）：

| 变异类 | 示例 |
|--------|------|
| 块内 CALL/WRITE 重排 | 同上——但本门 v2 SET 已正确放行（PASS by design） |
| 循环形态互转 | `while` ↔ `do-while` ↔ `for`（作者契约下本不应重写形态；Hex-Rays 伪代码已还原） |
| 编译器寄存器缓存的读次数/次序差 | CSE / register-caching → IDA 侧读次数 ≠ C++ 侧——READ advisory 已正确吸收 |
| 短路表达式的块拆分差异 | `a||b` 的单块 vs 两块 → STEP2 归一化已抹平 |
