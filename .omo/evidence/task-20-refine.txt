## Methodology
1. Read type_db.json → found 4 inferred_weak entries
2. Cross-referenced with purity_effects.json R/W map (all 4 had no direct purity data)
3. Performed byte-level inspection via IDA get_bytes for type evidence
4. Used IDA xref analysis to find referencing functions
5. Checked neighborhood purity (±256 bytes) for context
6. Applied rules from task spec:
   - No purity → per-address check → strong non-purity evidence → upgrade possible
   - Typeinfo strings = immutable (compiler-placed) → upgrade
   - Format strings = read-only (literal pool) → upgrade
   - Function table/dynamic data = mutable → keep weak
7. Scanned all 778 inferred_strong for read-only (R but no W) → added const to 4
